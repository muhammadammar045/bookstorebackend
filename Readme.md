# Bookstore Backend

This is the backend of the Bookstore application, built with Node.js, Express, and MongoDB. It provides APIs for managing books, user authentication, and file uploading using Multer and Cloudinary.

## Features

- **CRUD Operations**: Manage book details with create, read, update, and delete functionalities.
- **Pagination**: Handle large datasets by paginating book listings.
- **File Upload**: Upload book thumbnails using Multer and store images on Cloudinary.
- **User Authentication**: Secure login and registration for users.
- **RESTful API**: A well-structured RESTful API for interacting with the backend services.

## Technologies Used

- **Node.js**: A JavaScript runtime built on Chrome's V8 JavaScript engine.
- **Express**: A minimal and flexible Node.js web application framework.
- **MongoDB**: A NoSQL database for storing book and user data.
- **Mongoose**: An Object Data Modeling (ODM) library for MongoDB and Node.js.
- **Multer**: Middleware for handling `multipart/form-data`, used for file uploading.
- **Cloudinary**: A cloud-based image and video management service.

## Installation

To get started with the backend of the Bookstore application, follow these steps:

### Prerequisites

- Node.js and npm installed on your machine.
- MongoDB instance (local or cloud-based).
- Cloudinary account for image storage.
